# 老人端模拟数据说明

## 1. 文档目标

本文档说明当前老人端登录、注册、认领、首页状态展示所使用的本地模拟数据。

这些模拟数据和流程代码已经全部放在：

- `mobile/lib/features/elder/`

目录下，便于老人端开发时独立维护，不和其他模块混在一起。

## 3. 当前模拟登录与注册规则

当前老人端登录和注册都不依赖数据库和后端，统一使用以下规则：

- 登录手机号：任意 11 位手机号均可输入
- 注册手机号：任意 11 位手机号均可输入
- 演示环境建议密码：`123456`
- 登录时若密码不是 `123456`，会提示模拟密码规则

建议测试手机号：

- `13800138001`
- `13800138002`
- `13800138099`

其中：

- `13800138001` 登录后展示 1 位家属
- `13800138002` 登录后展示 2 位家属
- 其他手机号登录后展示 0 位家属

---

## 4. 当前模拟注册识别规则

老人端注册后，会根据手机号走本地模拟识别逻辑。

### 情况 1：手机号 `13800138001`
表示：

- 系统中已存在老人主体
- 老人姓名：张建国
- 已绑定家属数量：1
- 注册后进入“认领确认页”

### 情况 2：手机号 `13800138002`
表示：

- 系统中已存在老人主体
- 老人姓名：李秀英
- 已绑定家属数量：2
- 注册后进入“认领确认页”

### 情况 3：其他手机号
表示：

- 系统中不存在该老人主体
- 走“独立注册”流程
- 自动创建一个模拟老人主体
- 直接进入老人首页

---

## 5. 当前模拟流程用途

这套模拟逻辑主要用于：

- 没有数据库时先看页面效果
- 没有完整后端接口时先跑老人端流程
- 验证老人端登录、注册、认领、首页状态展示是否合理

当前模拟只用于老人端，不影响后续真实接口对接。

---

## 6. 后续替换建议

后续如果后端接口准备好了，可以按下面顺序逐步替换：

1. 用真实“老人登录接口”替换模拟登录
2. 用真实“老人注册接口”替换模拟注册
3. 用真实“老人主体查询接口”替换 `recognizeByPhone`
4. 用真实“老人认领接口”替换 `claimProfile`
5. 用真实“老人主体创建接口”替换 `createIndependentProfile`

这样可以保证老人端目录结构不变，只替换数据来源。
